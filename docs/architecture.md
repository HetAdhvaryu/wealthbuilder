# WealthBuilder – System Architecture

## 1. Overview

WealthBuilder is a modular trading research and signal generation platform designed to scan NSE F&O stocks and generate trading signals based on predefined strategies.

The system fetches market data, calculates indicators, runs multiple strategies, ranks signals, and sends alerts to users via Telegram.

Initial implementation uses **Yahoo Finance** as the data source and **Telegram** as the user interface.

The architecture is designed to be modular so that data providers, strategies, and user interfaces can be replaced or extended easily.

---

# 2. High-Level Architecture

```
Market Data Provider (Yahoo Finance)
        ↓
Data Fetcher Layer
        ↓
Indicator Engine
        ↓
Strategy Engine
        ↓
Signal Ranking Engine
        ↓
Alert System (Telegram)
        ↓
Signal Storage
```

Each component performs a single responsibility to maintain modularity and scalability.

---

# 3. System Components

## 3.1 Data Layer

**Module**

```
data/yahoo_fetcher.py
```

Responsibilities:

* Fetch market data from Yahoo Finance
* Retrieve OHLCV candles
* Convert raw responses into standardized DataFrame format
* Handle retry and error handling

Output format:

```
timestamp
open
high
low
close
volume
```

---

## 3.2 Universe Manager

**Module**

```
universe/fno_symbols.py
```

Responsibilities:

* Maintain list of F&O stocks
* Provide symbols to the scanner engine
* Future enhancement: automatically update F&O list from NSE

Example:

```
["RELIANCE", "ICICIBANK", "HDFCBANK", "TCS", "SBIN"]
```

---

## 3.3 Indicator Engine

**Module**

```
indicators/indicators.py
```

Responsibilities:

* Compute technical indicators required by strategies.

Example indicators:

* EMA
* VWAP
* Rolling High / Low
* Volume averages

Output:

Enhanced dataframe with indicator columns.

---

## 3.4 Strategy Engine

**Module**

```
strategies/
```

Each strategy is implemented as an independent module.

Example strategies:

```
breakout.py
volume_spike.py
momentum.py
```

Responsibilities:

* Evaluate market data
* Generate trade signals

Example signal format:

```
{
  "symbol": "ICICIBANK",
  "strategy": "breakout",
  "price": 1045,
  "score": 85
}
```

---

## 3.5 Scanner Engine

**Module**

```
scanner/engine.py
```

Responsibilities:

* Load trading universe
* Fetch market data
* Run indicator engine
* Execute all enabled strategies
* Collect signals
* Rank signals

Core loop:

```
for symbol in universe:

    data = fetch_data(symbol)

    indicators = compute_indicators(data)

    for strategy in strategies:

        signal = strategy.evaluate(data)

        if signal:
            store_signal(signal)
```

---

## 3.6 Signal Ranking Engine

Signals generated by strategies are ranked based on:

* Volume strength
* Momentum
* Breakout distance
* Strategy confidence

The ranking system ensures only the most relevant signals are sent to users.

Example output:

```
Top Signals

1. ICICIBANK – Breakout – Score 92
2. RELIANCE – Momentum – Score 88
3. SBIN – Volume Spike – Score 85
```

---

## 3.7 Alert System

**Module**

```
alerts/telegram_alert.py
```

Alerts are delivered through Telegram.

Telegram was chosen because:

* Instant notifications
* No UI development required
* Free and reliable

Example alert message:

```
🚨 Futures Signal

Symbol: ICICIBANK
Strategy: Breakout
Price: 1045
Volume Spike: 2.1x

Time: 11:15
```

---

## 3.8 Scheduler

**Module**

```
scheduler/run_scanner.py
```

Responsibilities:

* Run scanner engine at fixed intervals

Current configuration:

```
Every 15 minutes during market hours
```

Implementation uses Python scheduling libraries.

---

## 3.9 Signal Storage

**Module**

```
storage/signal_store.py
```

Responsibilities:

* Persist generated signals
* Track strategy performance
* Enable future backtesting

Stored fields:

```
timestamp
symbol
strategy
price
signal_score
```

Future versions may use databases such as SQLite or PostgreSQL.

# 4. Configuration System

Configuration file:

```
config/settings.json
```

Example:

```
{
  "timeframe": "15m",
  "symbols_limit": 20,
  "enabled_strategies": [
    "breakout",
    "volume_spike",
    "momentum"
  ]
}
```

This allows dynamic enabling or disabling of strategies without code changes.
